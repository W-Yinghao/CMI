# W0.4 — batch-size / weak-identification audit (FROZEN, before any W0.4 compute)

**Role.** Not a fifth result competing with W0.3/W0.5. It answers one reviewer question: *"is the W2
decision-prior harm just because the adaptation batch is too small?"* Frozen question:

> As `n` grows, does `π_J(n)` become more prevalence-like, and if so does that help or hurt the
> balanced-accuracy decision-prior branch?

**W0.3-informed prediction (frozen).** Exact per-draw decomposition on a FIXED evaluation set `E`:

    P_J(n) = [B_E(ρ_E) − B_E(Unif)]                 (metric-prior mismatch; constant negative anchor)
           + [B_E(ρ_A^draw(n)) − B_E(ρ_E)]          (sample / prevalence-transfer noise)
           + [B_E(π_J(n)) − B_E(ρ_A^draw(n))]       (π_J-deviation offset)

If larger `n` moves `π_J(n)` toward the non-uniform adaptation prevalence, the deviation offset shrinks, so
`P_J(n)` can become **MORE negative** even as `π_J` becomes a *better* prevalence estimate — because the
limiting prevalence prior is itself the wrong BA decision prior.

## Design (frozen)

- **Reuse** the frozen terminal W2 bundles (`p0_w2_bundles`, code_sig `763bf49d`, read-only); deterministic
  eval; real-subject-id fan-out (no bench index). Source seeds {0,1,2}.
- **`n ∈ {16, 32, 64, 128, 256}`**; **draws_per_unit = 5** (pre-declared), draw seed = `stable_hash(subj,n,draw)`.
- **Sampling = NATURAL-prevalence** (random `n` trials from the adaptation night, preserving the natural
  non-uniform stage distribution). *This SUPERSEDES the earlier "stratified where possible" note* — the
  mechanism is convergence of `π_J` to the non-uniform `ρ_A^full`, which stratified/balanced draws would
  destroy. Raw per-draw class counts `n_{A,y}` are recorded.
- **Fixed evaluation set** across all `n`/draws (the subject's evaluation night). `B_E(Unif)` and
  `B_E(ρ_E)` are therefore invariant across `n`/draws (anchors).
- **Two ρ_A definitions:** `ρ_A^draw(n)` = empirical prevalence of the sampled batch (used for the
  EXACT residual-zero decomposition); `ρ_A^full` = full adaptation-night prevalence (only for the
  convergence diagnostic `TV(π_J(n), ρ_A^full)`).
- **π_J(n) fit** by the joint EM on the sampled batch's embeddings (uniform weights).

## Prior floor / clipping (frozen BEFORE launch)

For small `n`, zero/near-zero stages are inevitable. Every decoded `B_E(·)` uses the **same predeclared
implementation prior transform: `log(clamp_min(π, 1e-12))`** (identical to the W0.3 prior-decomp, so
`B_E(π_J)` stays comparable). Applied uniformly to `ρ_A^draw(n)`, `ρ_E`, `π_J(n)`, `Unif`. The residual
check is computed AFTER this transform. Raw `min_y ρ_A^draw(n)` and missing-class counts are reported
separately (raw, un-clipped).

## Analyzer outputs (frozen), by `n`

Per-`n` (subject-cluster; **average draws AND seeds within subject before bootstrap** — draws are NOT
pseudo-subjects):
`P_J(n)`, `B_E(ρ_E)−B_E(Unif)`, `B_E(ρ_A^draw(n))−B_E(ρ_E)`, `B_E(π_J(n))−B_E(ρ_A^draw(n))`, and
`TV(π_J(n),ρ_A^draw(n))`, `TV(π_J(n),ρ_A^full)`, `min_y π_{J,y}(n)`, `H(π_J(n))`, `#{y: n_{A,y}=0}`.

**Primary statistical endpoint (frozen):** paired subject-cluster **endpoint contrast**
`Δ_{256−16} P_J = P_J(256) − P_J(16)` (and the same contrast for the π_J-deviation term and the TV
distances), 10k cluster bootstrap. NOT a separate CI at every `n`.

## QC sentinels (HARD GATES — run before trusting any W0.4 number)

1. real_subject_id fan-out coverage exactly matches the expected 75-subject set;
2. eval hash invariant across all `n`/draws for each subject/seed;
3. `B_E(Unif)` invariant across `n`/draws;
4. `B_E(ρ_E) − B_E(Unif)` invariant across `n`/draws (up to exact determinism);
5. decomposition residual = 0 (after the clipping transform);
6. `|B_E(π_J) − main P branch| = 0` where a main branch exists (N/A for sub-sampled draws; checked only
   if a full-adapt draw is included);
7. all prior floor/clipping choices recorded in the manifest;
8. missing-class counts reported, not silently excluded.

## Interpretation decision tree (FROZEN)

- `P_J(n)` **more negative** while `TV(π_J,ρ_A^draw)` **decreases** →
  "Larger adaptation batches improve prevalence identification but can worsen balanced-accuracy
  decision-prior use, because the limiting prevalence prior is itself the wrong BA decision prior."
- `P_J(n)` **flat**, metric-mismatch stays dominant →
  "The W2 prior-branch harm is already saturated by metric-prior mismatch; sample size changes prior
  quality but not the operational conclusion."
- small `n` **worse**, improves with `n` →
  "Small-batch prior instability adds a finite-sample penalty on top of metric-prior mismatch; larger `n`
  reduces that extra instability but does not make prevalence the BA decision prior."
- π_J-deviation term **positive at all `n`** → W0.3 wording: "π_J's deviation from adaptation prevalence
  partially offsets oracle-prevalence harm." (Do NOT say "flatter" without entropy/majorization support —
  `H(π_J(n))` is reported precisely to license or forbid that word.)
