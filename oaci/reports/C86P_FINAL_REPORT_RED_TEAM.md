# C86P Final Report Red Team

## Scope

The audit tested whether C86P prospectively defines a defensible active-testing
program without using new data or creating an executable real-data authority.

## Findings

1. **Protocol chronology: PASS after preserved corrections.** The main
   protocol preceded implementation. A shadow review then found an undefined
   low-budget bAcc path, which was prospectively closed with a fixed Jeffreys
   plug-in before registered execution. No C86 result existed.

2. **Untouched-cohort audit: PASS after preserved corrections.** The first
   catalog pass missed Yang2025's canonical 2C variant because the summary row
   combines 2C and 3C. A source-bound additive correction included
   `Yang2025_2C`. A later repository-access replay found prior local
   Dreyer2023 preprocessing evidence; fail-closed application of the original
   rule removed it from confirmation. Neither correction used performance.

3. **Completeness: PASS.** All 53 installed imagery catalog rows are present.
   Every catalog two-class row has a loader-source task audit. All and only
   `Brandl2020`, `Kumar2024`, and `Yang2025_2C` pass the final rule. Three
   untouched interfaces satisfy the at-least-two requirement.

4. **Development/confirmation separation: PASS.** C84/C85 and Dreyer2023 are
   development-only. Confirmation feedback cannot alter cohort inclusion,
   methods, hyperparameters, budgets, candidates, thresholds, or taxonomy.

5. **Fair query budget: PASS.** Budgets are total labels per target. P0 does
   not know class before query. P1 is visibly secondary and class-aware. FULL
   cannot generate an active-superiority claim.

6. **Estimator semantics: PASS with explicit limitation.** The sequential
   probability floor, LURE weights, budget-prefix replay, historical composite,
   Jeffreys small-prefix rule, and first-index action tie rule are fixed. Only
   linear LURE moments carry an unbiasedness claim. A4 confidence-set coverage
   is explicitly heuristic.

7. **Reference fidelity: PASS with adaptations disclosed.** A1--A4 bind
   primary literature, but their 81-candidate finite-pool adaptations are not
   represented as exact source replications. Future C86D may debug these fixed
   implementations; C86H cannot select among them from confirmation outcomes.

8. **Synthetic contract: PASS.** Eleven scenario names are backed by fixed
   generative laws, field dimensions, RNG/substream rules, production-entrypoint
   requirement, failure injection, and validation target. Registered draws and
   results remain zero; shadow helpers cannot publish a C86 result.

9. **Risk and inference: PASS.** Mean regret, worst target, CVaR, and
   epsilon-near-optimal selection are distinct. Target subject is the only
   scientific N; max-T, LOTO, level/panel/seed stability, taxonomy precedence,
   and no-pooling rules are prospective.

10. **Theory transfer: PASS.** T3 is limited to exact action collapse. T6
   motivates separate mean/tail reporting. T1/T2 do not identify unrestricted
   information value. T4/T5/T7 cannot be asserted empirically without their
   missing observation-law assumptions.

11. **Runtime boundary: PASS.** Static imports contain no MNE, MOABB runtime
    loader, Torch, project data loader, selector, inference, training, or GPU
    path. C86P created no C86 execution lock or authorization. Shadow tests did
    not run a registered C84 or new-data acquisition stream.

12. **Payload hygiene: PASS.** The C86P tracked protocol, code, tests, and table
    payload is under 200 KB before final reports. There are no generated EEG arrays, labels, model
    artifacts, scientific results, or external execution roots in Git.

## Residual Future-Stage Risks

Before any later access, the program must replay licenses, loader identities,
minimum per-target class support, candidate-field resource bounds, and absence
of prior access for each untouched interface. A split or support failure is
blocking and cannot trigger outcome-informed resplitting. C86L, C86D, C86C/F,
and C86H still need separate locks and authorization.
