# C57 - Information Boundary Formalism

Let each candidate-selection rule be measurable with respect to an availability class I0-I7. C57 treats this as empirical formalism over the frozen artifact universe, not as a minimax theorem.

- I0_random_or_tie: no row-specific candidate information beyond base rates -> not sufficient (random/tie comparator)
- I1_strict_source_observables: source risk, rank, leakage, and objective fields -> not reliable (available but insufficient)
- I2_source_plus_target_or_trajectory_keys: source fields plus target/trajectory identifiers or keys -> not sufficient (not original source-only DG)
- I3_target_unlabeled_transductive_geometry: unlabeled target geometry or confidence fields -> not sufficient (non-source-only)
- I4_target_grouped_zero_label_structure: target/trajectory grouped diagnostic structure -> not an available action rule (separate problem class)
- I5_split_label_or_few_label_calibration: target-label budget disjoint from evaluation labels -> open future (requires unavailable split-label cache)
- I6_target_label_diagnostic_content: target-label-derived diagnostic fields -> diagnostic only (not source-only)
- I7_same_label_endpoint_oracle: same-label endpoint scalar or joint margin -> diagnostic endpoint oracle (unavailable at selection time)

Bayes ceilings and endpoint oracles are diagnostic upper envelopes because they use target-label information outside the original source-only DG selection sigma-field.
